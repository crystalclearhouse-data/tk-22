---
description: savework workflow - used to synchronize state after manual execution
---

When the user says "savework", it indicates they have completed manual terminal executions and the system state is now updated.

1. Verify the results of the manual commands (e.g., check for new files, service status).
2. Update the `task.md` and `walkthrough.md` to reflect the progress.
3. Propose the next set of commands if applicable.
